# Getting Help

We're here to help! Here are the best ways to get support with ARK.

## Documentation

Start with the documentation:

- **[Quickstart](/quickstart)** - Installation and setup
- **[User Guide](/user-guide)** - Creating agents, teams, and queries
- **[Troubleshooting](./troubleshooting)** - Common issues and solutions
- **[FAQ](./faq)** - Frequently asked questions

## Support Channels

### GitHub Issues

For bugs, feature requests, and technical questions:

1. **Search existing issues**: Check if your question was already asked
2. **Check documentation**: Ensure the answer isn't in the docs
3. **Create new issue**: Provide details about your problem

[Open an issue on GitHub](https://github.com/mckinsey/agents-at-scale-ark/issues/new)

When creating an issue, include:
- Description of the problem
- Steps to reproduce
- Expected vs actual behavior
- ARK version
- Kubernetes version
- Relevant logs or error messages

### GitHub Discussions

For questions, ideas, and community discussions:

[Start a discussion](https://github.com/mckinsey/agents-at-scale-ark/discussions)

Perfect for:
- Asking "how do I..." questions
- Sharing use cases and experiences
- Discussing best practices
- Feature brainstorming

## Reporting Bugs

If you find a bug:

1. **Verify it's a bug**: Check documentation and FAQ first
2. **Check existing issues**: Search for similar reports
3. **Create detailed bug report**:
   - Clear title
   - Steps to reproduce
   - Expected behavior
   - Actual behavior
   - Environment details
   - Logs and error messages
   - Minimal reproduction case (if possible)

Use the [bug report template](https://github.com/mckinsey/agents-at-scale-ark/issues/new?template=bug_report.md) if available.

## Feature Requests

Have an idea for improvement?

1. **Search existing requests**: Check if it's already proposed
2. **Check roadmap**: See if it's planned
3. **Create feature request**:
   - Clear description
   - Use case / problem it solves
   - Proposed solution
   - Alternatives considered

Use the [feature request template](https://github.com/mckinsey/agents-at-scale-ark/issues/new?template=feature_request.md) if available.

## Security Issues

For security vulnerabilities, **do not** open a public issue. Instead:

1. Email security team (if contact available)
2. Use GitHub Security Advisory (if enabled)
3. Follow responsible disclosure practices

## Contributing

Want to help improve ARK? See [Contributing](./contributing) guide.

## Community Guidelines

- **Be respectful**: Treat others with kindness
- **Be helpful**: Share knowledge and help others
- **Be patient**: Remember we're all learning

## Other Resources

- **Stack Overflow**: Tag questions with `ark-k8s` or `agents-at-scale` (if community forms)
- **Slack/Discord**: Join community channels (if available)
- **Blog**: Check for updates and tutorials
- **YouTube**: Watch video tutorials (if available)

## Still Need Help?

If you've tried the above and still need assistance:

1. **Check Troubleshooting**: [Troubleshooting Guide](./troubleshooting)
2. **Review FAQ**: [Frequently Asked Questions](./faq)
3. **Search Issues**: Look through closed issues for solutions
4. **Ask Community**: Post in Discussions
5. **Open Issue**: If nothing helps, create a detailed issue

## Tips for Getting Help Faster

1. **Provide context**: Include your setup, versions, and environment
2. **Include logs**: Share relevant logs and error messages
3. **Minimal reproduction**: Create the smallest example that demonstrates the issue
4. **Be specific**: Describe exactly what you tried and what happened
5. **Be patient**: Maintainers are volunteers with limited time

Thank you for using ARK! We appreciate your feedback and contributions.
